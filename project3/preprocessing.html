Implementation of Preprocessing

Preprocessing is the term used to describe the translation of developer-use CSS into web-use CSS. There are a few ways to implement preprocessing into CSS. The two major CSS preprocessors are SASS and LESS. After choosing one, you must install the files onto your computer. Once this is complete, you are mostly ready to begin coding with it. A VS Code plugin called "Live SASS Compile" allows for the CSS to automatically compile when I save my .scss (SASS) files. By using this plugin, I was able to save time by not needing to manually type commands into the console.

Mechanisms of SASS Preprocessing

From a development perspective, using SASS or LESS to develop CSS for their websties allows for a quicker and more flexible workflow. Both SASS and traditional CSS have very similar syntax, but SASS introduces new ways of typing the same thing. One can do this through the use of nesting and by use of variables. Traditionally, you would need to create a new line of CSS for the base element, then new lines for every form of that element. A new line would need to be created for diferent states, such as when the element is being hovered over. Similarly, if you wanted to change the CSS of an element within a certain element, you'd need to list it all out specifically. In SASS, you can simply add new CSS within that element by using "&" before the next level, such as :hover or a nested element.

Variables

Setting variables allows for long-term flexibility if overarching changes need to be made. For example, if you want to set a brand shade of red, you could make it a variable like this: $brand-red. When you change the value of the variable, it will carry those changes all throughout the code where it is being used.

Mixin and Include

Using "@mixin" allows multiple lines of CSS to be saved and used in future code. These mixin styles can then be called later and used with "@include". In many ways, you could think of a mixin as a link to include many styles.

Functions

Functions can be used in numerous ways throughout SASS. The function is first defined like this "function()". Within the parenthesis, the developer can input variables for processing. If there is more than one variable, each will be separated by a comma. The developer then defines the output by declaring how the variables will be processed. In the compiled CSS, the function will be replaced by the output of the function.

Operators

Just like standard math, operators in SASS include plus (+), minus (-), multiply (*), and divide (/). Additionnally, there is an unusual operator called "modulus." It is represented by a percentage sign (%). Modulus divides indegers as many times as possible and gives the remainder. For example, 7 % 3 = 1. Three goes into seven twice, and there is one left. Operators also include these: equal to (==), not equal to (!=), greater than (>), greater than or equal to (>=), and the less than alternates. There are also the boolean terms: "and", "or" and "not".

Nesting

Nesting allows developers to save time by nesting CSS for sub-elements or pseudo-classes directly into their parent element's CSS. Traditionally you'd need to create a whole new line of CSS selecting the full route from the parent to its sub-elements. By directly nesting within the parent CSS, we can save time while also promoting readability.

Client-Side Vs Server-Side Preprocessing

Both client-side and server-side preprocessing have their advantages and disadvantages. Here are the differences:

Client-Side

This processing happens within a user's web browser. Typically, these include processing HTML, CSS, and JavaScript.

Server-Side

The server-side processing happens on the servers that are hosting the website. These processes are more interactive and responsive to user activity and requests.
